# SIMSOFT Installer Guide

The Windows installer is generated by Tauri from `src-tauri/tauri.conf.json`.

## Installer Types

- `SIMSOFT_Setup.exe`: NSIS installer
- `SIMSOFT.msi`: MSI installer

Both are copied to `release/` by `scripts/build_release.ps1`.

## End-User Installation

Operators should only need to:

1. Run `SIMSOFT_Setup.exe` or `SIMSOFT.msi`.
2. Open SIMSOFT Posting from the Start Menu or desktop shortcut if enabled by the installer.
3. Sign in with Google.
4. Request or receive access from the admin.
5. Use the app.

No Python, Node.js, Rust, Git, VS Code, or terminal setup is required on operator PCs.

## Runtime Locations

Installed app data:

```text
%LOCALAPPDATA%\SIMSOFT Posting\data
%LOCALAPPDATA%\SIMSOFT Posting\logs
```

Portable app data:

```text
release/portable/SIMSOFT Posting/data
release/portable/SIMSOFT Posting/logs
release/portable/SIMSOFT Posting/storage
```

## Credentials

Production Google credential files are deployment files:

```text
config/service_account.json
config/oauth_client.json
```

Do not commit real credential files. Use shared deployment procedures or a secure admin-managed installation path.

If the installer is built with real credentials present, those credentials are bundled into the release. Treat installer artifacts as company-confidential files.
